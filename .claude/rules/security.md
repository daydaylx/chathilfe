# Security and privacy rules

- Follow `docs/PRIVACY_SECURITY.md` before changing data handling.
- Never commit real secrets.
- Do not store user texts, generated suggestions, retry instructions, or clipboard history.
- Do not log API keys, user texts, clipboard text, retry instructions, or generated replies.
- Keep API-key handling build-time only for the MVP.
- Keep the app free of history, profiles, memory, analytics, and tracking.
